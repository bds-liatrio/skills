# Clarifying question template

Copy this block for every clarifying question (file + chat). Do not invent a
different shape.

Rules:

- **A–C** — concrete alternatives grounded in the repo / issue when possible
- **D** — always `Other` (user free-texts); never omit D
- Every option includes a one-line rationale after `—`
- **Recommended** — exactly one of A|B|C|D plus a one-line why
- **Answer** — `_pending_` until the user replies; then `A` / `B` / `C` / `D — <text>`

```markdown
### Q<n> — <seal-checklist gap>
Why: <what would force an implementing agent to guess>
- **A)** <option> — <rationale>
- **B)** <option> — <rationale>
- **C)** <option> — <rationale>
- **D)** Other — user specifies
Recommended: <A|B|C|D> — <one-line why this is the default>
Answer: _pending_
```
